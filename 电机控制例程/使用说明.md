---

## 方式一：作为模块导入（推荐，适合程序自动调用）

你的上位机程序直接 `import` 这个文件，构造参数后调用 `build_usb_packet()`，
拿到 `bytes` 对象后写入 USB 设备节点即可。

### 示例代码

```python
import serial                          # pip install pyserial
from usb_packet_builder import build_usb_packet

# 构造12个电机参数（每个电机独立）
motors = [
    {'kp': 50.0, 'kd': 0.5, 'pos':  1.0, 'vel': 0.0, 'tor': 0.0},  # 电机1
    {'kp': 50.0, 'kd': 0.5, 'pos': -1.0, 'vel': 0.0, 'tor': 0.0},  # 电机2
    {'kp': 30.0, 'kd': 1.0, 'pos':  0.5, 'vel': 0.5, 'tor': 2.0},  # 电机3
    # ... 共12个，每个参数可以不同
]

# 打包为164字节
packet = build_usb_packet(motors, sequence=0)

# 发送到 USB CDC 设备（Linux 下通常是 /dev/ttyACM0 或 /dev/ttyUSB0）
with serial.Serial('/dev/ttyACM0', baudrate=115200, timeout=1) as ser:
    ser.write(packet)
```

### 参数范围

| 参数 | 范围 | 单位 |
|------|------|------|
| kp | 0 ~ 500 | — |
| kd | 0 ~ 5 | — |
| pos | -12.5 ~ +12.5 | rad |
| vel | -18 ~ +18 | rad/s |
| tor | -30 ~ +30 | Nm |

## 方式二：持续控制循环（上位机实时控制）

上位机程序在控制循环里周期性调用，每次用最新期望值打包发送：

```python
import time
import serial
from usb_packet_builder import build_usb_packet

ser = serial.Serial('/dev/ttyACM0', baudrate=115200, timeout=1)
sequence = 0

while True:
    # 从你的控制算法获取最新期望值（此处替换为你的实际来源）
    motors = get_desired_motor_params()   # 你自己的函数

    packet = build_usb_packet(motors, sequence=sequence)
    ser.write(packet)

    sequence = (sequence + 1) % 256      # 序列号循环递增，用于丢包检测
    time.sleep(0.01)                      # 100Hz 控制频率，按需调整
```

## 方式三：命令行 JSON 传参（适合脚本调用）

如果你的上位机是其他语言（C++/ROS等），可以把参数序列化为 JSON 通过命令行传给本脚本，
脚本打包后直接写入 USB 设备，不需要 Python 进程常驻。

### 第一步：修改脚本支持 JSON 输入

在 `usb_packet_builder.py` 末尾的 `if __name__ == '__main__':` 块中，
增加以下逻辑（已在下方给出完整替换内容）：

```python
import sys, json

if __name__ == '__main__':
    if len(sys.argv) == 2:
        # 命令行传入 JSON 字符串
        motors = json.loads(sys.argv[1])
        pkt = build_usb_packet(motors, sequence=0)
        # 直接写入 USB 设备
        with open('/dev/ttyACM0', 'wb') as f:
            f.write(pkt)
    else:
        # 无参数时走交互式输入
        motors = input_motors_interactive()
        pkt = build_usb_packet(motors, sequence=0)
        print_usb_packet(pkt)
        print(' '.join(f'{b:02X}' for b in pkt))
```

## Linux USB 设备节点

STM32 USB CDC 在 Linux 下通常识别为：

```
/dev/ttyACM0
```

插入后用以下命令确认：

```bash
ls /dev/ttyACM*
# 或
dmesg | tail -20
```

如果提示权限不足：

```bash
sudo chmod 666 /dev/ttyACM0
# 或永久加入 dialout 组（重启后生效）
sudo usermod -aG dialout $USER
```

## USB 包结构速查

```
Byte   0      : 帧头 0xAA
Byte   1      : 命令类型 0x10
Byte   2~3    : 数据长度大端 0x00 0x60 (=96)
Byte   4      : 序列号 0~255
Byte   5      : 保留 0x00
Byte   6~101  : 12个电机×8字节控制数据
Byte 102~109  : 保留 0x00×8
Byte 110      : XOR校验和（Byte 0~109 异或）
Byte 111      : 帧尾 0x55
Byte 112~163  : 填充 0x00×52
共 164 字节
```
