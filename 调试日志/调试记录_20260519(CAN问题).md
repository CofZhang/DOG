# CAN 发送帧间延迟问题

**日期**：2026-05-19
**芯片**：STM32H743（主控）
**文件**：`DOG_Master/Core/Src/system_control.c`

---

## 问题描述

主控通过 FDCAN1 控制电机1-3（ID=0x01/0x02/0x03），三个电机串联在同一条 CAN 总线上。

现象：
- 单独接 ID3 电机：正常旋转
- 接 ID3 + ID2：正常旋转
- 三个电机全部接上：ID1、ID2 正常运行，**ID3 不动**（上电后抖动一下随即停止）

---

## 排查过程

| 排查项 | 结果 |
|--------|------|
| CAN 总线层（ECR/PSR 寄存器） | ECR=0，PSR=0x708，总线完全正常，无 Bus-Off |
| Tx FIFO 是否满 | `HAL_FDCAN_GetTxFifoFreeLevel` 一直返回3，FIFO 未满 |
| ID3 帧是否成功进入 FIFO | `g_can_tx_ok[2]` 持续累加，发送全部成功，无失败 |
| 电机 ID 配置 | 三个电机 ID 各不相同，排除 ID 冲突 |
| 接线顺序调换 | 无论哪个电机接在末端，始终是 ID3 不动，与物理位置无关 |
| 单步调试 | 单步时三个电机轮流正常运行，全速运行时 ID3 不动 |
| 断点暂停后取消断点 | 三个电机同时运行，但只持续一包数据的时间 |
| 控制参数检查 | `g_motor_params[2]` 的 kp/kd/velocity 与其他电机完全一致 |

**关键发现**：在 `HAL_FDCAN_AddMessageToTxFifoQ` 调用前加 `HAL_Delay(1)`，三个电机全部正常运行。说明电机需要帧间至少 1ms 间隔才能正常响应，连续发送三帧时 ID3 来不及处理。

---

## 根本原因

EC-A6408-P2-25 电机对连续接收的 CAN 帧有最小间隔要求。主控在一次主循环中连续塞入三帧（ID1→ID2→ID3），帧间间隔极短（微秒级），ID3 的帧到达时电机内部尚未处理完前一帧，导致 ID3 帧被丢弃。

单步调试时帧间有足够的人工暂停时间，所以正常。

---

## 解决方案演进

### 方案一：HAL_GetTick 忙等（已废弃）

在 FDCAN 发送循环中，每帧发送前等待 SysTick 跳变到下一毫秒：

```c
static uint32_t s_last_tx_tick = 0;

for (int i = 0; i < 3; i++) {
    while (HAL_GetTick() == s_last_tx_tick);
    s_last_tx_tick = HAL_GetTick();
    // ... 发送帧
}
```

问题：控制周期 5ms 时，三帧共占用 ~2ms（40% 开销），影响主循环响应。

---

### 方案二：TIM6 定时器中断（最终方案）

将三帧 CAN 发送移入 TIM6 中断，每 1ms 发一帧，主循环完全不阻塞。

**CubeMX 配置（TIM6）**：

| 参数 | 值 |
|------|-----|
| Activated | 勾选 |
| Prescaler | 239 |
| Counter Period | 999 |
| auto-reload preload | Enable |
| NVIC - TIM6 global interrupt | 勾选，Priority = 5 |

> APB1 timer clock = 240MHz，240MHz ÷ 240 ÷ 1000 = 1kHz（1ms 中断）

**`system_control.c` 逻辑**：

- `System_Control_Init()` 调用 `HAL_TIM_Base_Start_IT(&htim6)` 启动定时器
- `System_ProcessMotorControl()` 解析 USB 包后，预打包三帧数据到 `g_can_tx_data[3][8]`，设置 `g_can_tx_pending = 1`
- `HAL_TIM_PeriodElapsedCallback()` 每 1ms 触发，依次发送电机1→2→3，发完后清除 `g_can_tx_pending`

```c
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance != TIM6) return;
    if (!g_can_tx_pending) return;

    uint8_t i = g_can_tx_index;
    // 构造 tx_header，发送 g_can_tx_data[i]
    HAL_FDCAN_AddMessageToTxFifoQ(&hfdcan1, &tx_header, g_can_tx_data[i]);

    g_can_tx_index++;
    if (g_can_tx_index >= 3) {
        g_can_tx_pending = 0;
    }
}
```

**效果**：5ms 控制周期内主循环零阻塞，三帧在后台 3ms 内由中断完成发送。

---

## 影响评估

- 主循环不再有任何等待，USB 接收和 SPI 处理不受影响
- 控制周期可缩短至 5ms 甚至更低
- 每个电机仍以完整的 USB 频率收到指令，不降低控制频率
