# 调试记录 2026-05-19

## 问题描述

主控 `g_motor_feedback` 数组中，电机1-3的反馈数据正常，电机4-12（从控管理）的反馈数据全为0，无法收到。

## 现象

- 主控 `g_spi_rx_buffer` 全为0
- 从控 FDCAN 能正常收到电机反馈（`motorFeedbackRaw` 有数据）
- 从控 `spiFeedbackPacket` 打包正常（有数据）
- SPI 控制链路正常（电机4-12可以被控制）

## 排查过程

**断点1**：`DOG_Slaver/Core/Src/slave_controller.c:322`
- `memcpy(g_slaveCtrl.motorFeedbackRaw[i], rxData, 8)`
- 结果：正常，FDCAN 收到了各电机反馈，`rxData` 有数据

**断点2**：`DOG_Slaver/Core/Src/slave_controller.c:227`
- `memcpy(feedback->motorFeedback, ...)`
- 结果：正常，`motorFeedbackRaw` 有数据，反馈包打包正确

**断点3**：`DOG_Master/Core/Src/u_spi.c:124`
- `g_spi_rx_buffer[0]` 检查
- 结果：全为0，说明从机 MISO 根本没有发出有效数据

## 根本原因

### 第一次尝试（错误方向）

怀疑是 DMA CMAR 寄存器在 DMA 使能时无法修改，尝试在修改 CMAR 前先禁用 DMA 通道：

```c
hspi1.hdmatx->Instance->CCR &= ~DMA_CCR_EN;
while (hspi1.hdmatx->Instance->CCR & DMA_CCR_EN);
hspi1.hdmatx->Instance->CMAR = (uint32_t)&g_slaveCtrl.spiFeedbackPacket;
hspi1.hdmatx->Instance->CCR |= DMA_CCR_MINC;
hspi1.hdmatx->Instance->CCR |= DMA_CCR_EN;
```

结果：问题依旧，`g_spi_rx_buffer` 仍全为0。

### 真正原因

查看 STM32G4 HAL 源码 `stm32g4xx_hal_spi.c:2041`：

```c
// HAL_SPI_Receive_DMA 在2线全双工模式下的实现：
return HAL_SPI_TransmitReceive_DMA(hspi, pData, pData, Size);
```

**`HAL_SPI_Receive_DMA` 在全双工模式下，内部直接调用 `HAL_SPI_TransmitReceive_DMA`，且 TX 和 RX 指向同一个缓冲区（`spiRxPacket`）。**

HAL 用 `HAL_DMA_Start_IT` 启动 TX DMA 后，CMAR 已经被锁定为 `spiRxPacket` 的地址。后续无论如何重定向 CMAR，HAL 在下一次调用 `HAL_SPI_Receive_DMA` 时都会重新把 TX DMA 指回 `spiRxPacket`。

因此从机 MISO 发出的是 `spiRxPacket`（主控发来的控制数据的回声），而不是反馈包。主控收到的是自己发出去的数据的镜像，校验不通过，`g_spi_rx_buffer` 被判定为无效，反馈不更新。

控制链路之所以正常，是因为控制数据走的是 **主控TX → 从机RX**，与从机TX无关。

## 解决方案

放弃 `HAL_SPI_Receive_DMA` + CMAR hack，改用 `HAL_SPI_TransmitReceive_DMA`，明确指定 TX 缓冲区为反馈包：

```c
HAL_SPI_TransmitReceive_DMA(&hspi1,
    (uint8_t*)&g_slaveCtrl.spiFeedbackPacket,  // TX: 反馈包
    (uint8_t*)&g_slaveCtrl.spiRxPacket,        // RX: 控制包
    spi_length);
```

同时将回调函数从 `HAL_SPI_RxCpltCallback` 改为 `HAL_SPI_TxRxCpltCallback`（全双工DMA完成触发的是TxRx回调）。

## 修改文件

`DOG_Slaver/Core/Src/slave_controller.c`

1. `SlaveController_Init`：`HAL_SPI_Receive_DMA` → `HAL_SPI_TransmitReceive_DMA`，删除 CMAR 重定向代码
2. `SlaveController_Process` 错误恢复分支：同上
3. `SlaveController_Process` 正常重启分支：同上
4. 回调：`HAL_SPI_RxCpltCallback` → `HAL_SPI_TxRxCpltCallback`

## 结果

修改后主控 `g_spi_rx_buffer[0]` = `0xAA`，12个电机反馈数据全部正常收到。
