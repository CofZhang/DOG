## 一、协议参数范围错误

### 问题描述
`motor.h` 和 `motor.c` 中的物理值范围与官方例程不符，KP 和 KD 范围互换，位置/速度/扭矩范围也错误。

### 对比

| 参数 | 错误值 | 正确值（官方） |
|------|--------|--------------|
| KP 范围 | 0 ~ 5 | 0 ~ 500 |
| KD 范围 | 0 ~ 50 | 0 ~ 5 |
| 位置范围 | ±60 rad | ±12.5 rad |
| 速度范围 | ±60 rad/s | ±18 rad/s |
| 扭矩范围 | ±30 Nm | ±30 Nm |

### 修复
- `motor.h`：修正所有范围宏定义
- `motor.c`：换算公式改为官方 `float_to_uint` 通用公式：
  ```
  raw = (phys - phys_min) / (phys_max - phys_min) * (2^bits - 1)
  ```
- `MotorFeedback` 结构体：去掉 `coil_temp` + `mos_temp` 两个字段，改为单个 `temperature`
- 反馈电流换算：`(raw/4095)*124-62` → `raw*60/4095-30`（范围 -30~+30 A）

## 二、g_motor_params 数据不更新问题

### 现象
- `g_rx_buffer` 收到数据 ✓
- `packet_buffer` 有数据 ✓
- `g_motor_params` 全是 0 ✗

### 调试方法

#### 断点位置
```
usbcol.c - Protocol_ReadPacket() 函数内
第416行：if (result == PROTOCOL_STATE_COMPLETE)
```

#### 观测内容
在 Keil Watch 窗口输入：
```
usbcol\g_protocol_stats
```
查看哪个计数器不为0：
- `header_errors`：帧头不是 0xAA（正常容错，可忽略）
- `footer_errors`：Byte 111 不是 0x55
- `checksum_errors`：校验和不匹配
- `length_errors`：数据长度字段不是 96

#### 根本原因1：帧尾和校验和偏移量错误
代码用 `USB_PKG_TOTAL_LEN-1`（Byte 163）检查帧尾，用 `USB_PKG_TOTAL_LEN-2`（Byte 162）读校验和，但实际帧尾在 Byte 111，校验和在 Byte 110。

**修复**：新增宏定义，所有相关位置改用固定偏移：
```c
#define USB_PKG_CHECKSUM_OFFSET  110
#define USB_PKG_FOOTER_OFFSET    111
#define USB_PKG_CHECKSUM_LEN     110
```

涉及文件：`usbcol.c` 中的 `Protocol_ParseControlPkg`、`Protocol_CheckPacket`、`Protocol_VerifyChecksum`、`Protocol_PackFeedbackPkg`、`Protocol_PackControlPkg`。

#### 根本原因2：packet_buffer 在栈上导致栈溢出
`Protocol_ReadPacket` 中 `packet_buffer[164]` 的 `static` 关键字被误删，变成栈上局部变量。栈只有 1024 字节，164 字节的局部数组加上调用链导致栈溢出，`out_pkg` 指针被破坏。

**修复**：
```c
// 改回 static
static uint8_t packet_buffer[USB_PKG_TOTAL_LEN];
```
同时将 `startup_stm32h743xx.s` 中栈大小从 `0x400` 扩大到 `0x1000`。

## 三、校验和计算错误

### 问题描述
手动计算校验和出错，导致发送的数据包校验和字段不正确，STM32 解析失败。

### 调试方法

#### 断点位置
```
usbcol.c - Protocol_ParseControlPkg()
第90行：if (calc_checksum != pkt_checksum)
```

#### 观测内容
在 Watch 窗口直接观测局部变量：
```
calc_checksum    ← STM32 计算出的校验和
pkt_checksum     ← 数据包 Byte 110 的值
```
或鼠标悬停在变量名上查看。

### 校验和计算规则
对 Byte 0~109（共110字节）逐字节异或：
```python
checksum = 0
for i in range(110):
    checksum ^= data[i]
```

### 正确数据包示例（5rad/s速度，kd=2）
```
AA 10 00 60 00 00 00 00 CC 7F FF A3 97 FF 00 00 00 7F FF 7F F7 FF ...（×11）... 00 00 00 00 00 00 00 00 00 00 AA 55 00...00
```
Byte 110 = `0xAA`，Byte 111 = `0x55`

## 四、Keil 无法观测 static 变量

### 问题描述
`g_rx_buffer`、`packet_buffer`、`g_motor_params` 在 Watch 窗口直接输入变量名找不到。

### 原因
三个变量都是 `static`，符号可见性被限制在编译单元内，不导出到全局符号表。

### 解决方法

**方法1（推荐）：加模块限定符**
```
usbcol\g_rx_buffer
usbcol\g_protocol_stats
system_control\g_motor_params
```

**方法2：在变量所在文件打断点**
程序停在 `usbcol.c` 的断点时，Watch 窗口可直接输入变量名，无需限定符。

**方法3：Memory 窗口**
先用 `&usbcol\g_rx_buffer` 获取地址，再去 Memory 窗口查看原始内存。

## 五、数据包格式说明

### 打包公式（官方位操作）
```c
Data[0] = 0x00 | (kp_int >> 7)
Data[1] = ((kp_int & 0x7F) << 1) | ((kd_int & 0x100) >> 8)
Data[2] = kd_int & 0xFF
Data[3] = pos_int >> 8
Data[4] = pos_int & 0xFF
Data[5] = spd_int >> 4
Data[6] = (spd_int & 0x0F) << 4 | (tor_int >> 8)
Data[7] = tor_int & 0xFF
```

### 安全值（零位、零速、零力矩）
```
kp=0, kd=0, pos=0rad, vel=0rad/s, tor=0Nm
→ pos_raw=0x7FFF, vel_raw=0x7FF, tor_raw=0x7FF
→ 8字节：00 00 00 7F FF 7F F7 FF
```

### 注意事项
- **kd 不能为 0**，否则速度模式下输出扭矩为0，电机不动
- 速度模式建议 kd 从 1.0~2.0 开始调试
- 扭矩原始值为0对应物理值 -30Nm，不是零扭矩

## 六、数据流调用链

```
USB中断
  └─ CDC_Receive_FS()                    [usbd_cdc_if.c]
       └─ Protocol_ProcessRxData()        [usbcol.c] → 写入 g_rx_buffer

主循环 while(1)
  └─ System_Control_Process()            [system_control.c]
       └─ Protocol_HasCompletePacket()    [usbcol.c] → 检查帧头+长度
            └─ System_ProcessMotorControl()
                 └─ Protocol_ReadPacket(g_motor_params, ...)  [usbcol.c]
                      └─ packet_buffer ← 从 g_rx_buffer 复制164字节
                           └─ Protocol_ParseControlPkg()
                                └─ Motor_UnpackControlData() → 写入 g_motor_params
                 └─ Motor_PackControlData() → 打包CAN数据
                 └─ HAL_FDCAN_AddMessageToTxFifoQ() → 发送给电机
```
